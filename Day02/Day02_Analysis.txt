[ systemctl status httpd 분석 ]
1. 아파치 Http 서버 
2. Loaded : 서버스가 설치되어있는 위치 : (자동 활성화 상태) enable / disable;  preset : 배포판 권장 기본값(자동 활성화 X)
3. Active : 실행상태 (running / stop) 실행 날짜 시간 등 표시
4. Invocation : systemd가 서비스 실행 단위를 추적하기 위해 생성하는 실행 세션 ID (즉, 실행 자체를 식별)
5. Docs : man:httpd.service(8) --> man 8 httpd.service 를 치면 사용 방법과 설명서를 보여줌 
	    즉, 해당 서비스에 대한 자세한 사용 설명서가 어디에 있는지 알려주는 안내 표지판임

6. Main PID: httpd의 프로세스 ID
7. Status : 아파치 웹서버가 현재 실시간으로 처리하고 있는 웹 트래픽 및 접속 처리 상태 (요약)
	      Idle/Busy workers 100/0; -> 대기하고 있는 프로세스 / 일하고 있는 프로세스
	      이것을 통해 너무 많은 요청이 들어온지에 대한 트래픽 과부화를 체크하고 확인할 수 있음

8. Tasks : 현재 아파치 웹 서버가 내부적으로 쪼개어 가동하고 있는 프로세스와 스레드의 합계 --> 프로세스 + 스레드
             사용자가 많이 접속할수록 이 숫자가 늘어남 (limit 는 최대 제한) (항상 성립은 아님)
 
		ㄴ> 왜 여러개로 쪼개어 가동할까?
		ㄴ> 동시에 접속하는 수많은 사용자를 효율적으로 처리하기 위해 여러 개의 자식 프로세스나 스레드를 복제해서
                      나누어 일하는 방식(MPM : Multi-Processing Module)을 사용한다.
		ㄴ> 웹 사이트 동시 접속자가 limit 한계치까지 치솟게 되면 새로운 접속 요청을 거부하게 되는데 이때 관리자가
  		      이 한계치를 올려주는 설정을 해주어야 한다.

9. Memory : 아파치 웹 서버가 사용하고 있는 메모리 크기
10. CPU : 아파치 웹 서버가 켜진 순간부터 지속적으로 CPU 자원을 사용한 시간 
11. CGgroup : 이 서비스가 관리하고 있는 모든 프로세스들의 계보와 명단을 트리구조로 보여주는 항목
		   맨 위 PID가 대장 프로세스 이고 그 아래 PID들은 대장 프로세스가 복제해 낸 자식 프로세스들이다.
		  
			<리눅스가 CGroup을 사용하는 이유>
			만약 아파치 웹 서버를 종료할 때, 자식 프로세스들이 남으면 서버가 지저분해 진다.
			그래서 이 CGroup이라는 바구니를 통째로 뒤엎어 버리는 것이 편하기 때문에 리눅스는
			웹 서버 종료시 바구니 안에 든 모든 자식 프로세스들을 단 하나의 잔당도 남기지 않고 일괄 종료시킨다
			또한, 자원의 제한을 걸어 CPU 과부화를 막을 수 있다




[ ps aux | grep httpd ]
1. 왜 여러개가 실행 중인가?
 	ㄴ> 이 경우 아까 위의 설명했던 것과 마찬가지로 여러개의 요청이 들어왔을 때 이것을 효율적으로 관리하기 위해서
	      대장 PID가 여러개의 자식 PID를 만들어 내어 관리한다. 이로 인해 사용자들은 좀더 빠르고 쾌적하게 아파치 웹 
  	      서비스를 이용할 수 있게 된다.

2. 부모 프로세스는? 
	ㄴ> 12901 : 대장 프로세스로서 아래 자식 프로세스들의 원본인 부모이다.

3. 자식 프레세스는?
	ㄴ> 12902 ~ 13915 : 176개의 복제품으로써 원본의 역할을 분담하여 효율적으로 관리하는 자식 프로세스들이다.





[ sudo ss -tulpin ] - 포트 보기
-> 현재 시스템에서 네트워크 열려 있는 포트와 이를 사용하는 프로세스 명단을 출력한 결과
-> -t : TCP 포트 출럭
-> -u : UDP 포트 출력
-> -l : 연결을 대기중인(Listen) 포트 출력
-> -p : 포트를 사용하는 프로세스 이름과 PID를 출력 (이것 때문에 sudo가 필요함)
-> -i : TCP 내부 성능 지표 (cubic cwn:10 같은 데이터) 출력
-> -n : 서비스 이름(http, ssh 등) 대신 숫자 포트 번호(80, 22)로 출력

1. 80포트는 누가 사용하고 있는가?
	ㄴ> httpd 즉, 아파치 웹 서버 데몬이 사용하고 있다. 

2. 왜 80인가?
	ㄴ> 포트번호 80은 주로 웹 사이트인 http가 사용하는 포트 번호이기 때문이다

3. 443은 무엇인가?
	ㄴ> 443은 https로, http에 보안 기능을 추가한 것이다.
	ㄴ> 또한 통신 내용 전체가 암호화 된다.



[ cd /var/www/html ]
1. 웹페이지는 어디에 저장되는가? : /var/www/html 
	ㄴ> 리눅스 아파치 웹 서버의 기본 웹 문서 루트 디렉터리가 바로 (기본값이)
		/var/www/html이다. 사용자가 인터넷 브라우저 주소창에 서버 IP를 입력하면, 시스템은 이 폴더 내부를
		가장 먼저 탐색한다 (즉, 아파치가 DocumentRoot 기준으로 찾고 그것이 /var/www/html 인 것)

2. 왜 브라우저가 그 파일을 보여주는가?
	ㄴ> 아파치 설정 파일(httpd.conf)의 DirectoryIndex 설정과 웹 서버의 응답 메커니즘 때문이다.

		1) 기본 파일 우선 인식 : 사용자가 파일명을 주소창에 직접 입력하지 않고 IP만 치고 들어오면,
						웹 서버는 해당 폴더에서 index.html 또는 index.php 같은 기본 대표 파일을
						자동으로 찾아서 읽도록 설정되어 있다.

		2) 80번 포트 상시 대기: httpd 프로세스가 80번 포트를 열고 항상 대기하고 있다가, 브라우저가 보낸 요청을
						수신한다

		3) HTML 코드 해석 : 웹 서버가 /var/www/html 에 있는 파일의 내용을 브라우저에게 그대로 던져주면
					  사용자의 웹 브라우저가 그 코드를 보기 좋은 화면으로 해석(렌더링)하여 보여준다
