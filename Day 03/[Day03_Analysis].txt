[Day03 명령어 분석]

1. ps (Process Status) : 현재 실행 중인 프로세스 확인

2. ps aux 
	ㄴ> a : 모든 사용자 프로세스
	ㄴ> u : 상세 정보 출력
	ㄴ> x : 터미널에 연결되지 않은 프로세스까지 출력

	=> 즉, 현재 컴퓨터에서 실행중인 거의 모든 프로그램 출력


< 복습 >
왜 아파치는 PID가 여러개인가?
	ㄴ> 하나의 부모 프로세스로 모든 요청을 처리하는 것 보다 자식 프로세스 
		즉, 복제품들을 만들어 쪼개어 요청을 분산 처리하는 것이 더 효율적이기 때문에
		이러한 방식을 채택하였다. 



[ pstree 이해하기 ]
1. pstree : 말그대로 프로세스를 트리 형식으로 정렬시키는 명령어이다

2. pstree -p : 프로세스 트리를 보여줄때 각 프로세스의 PID도 같이 표시한다. 




[ ip에 관하여 ]
1. ip a (ip address) : 현재 사용중인 컴퓨터에 장착된 모든 네트워크 카드와 할당된 IP 주소 정보를
			  화면에 보여달라는 명령어

2.  IP는 왜 필요한가?
	ㄴ> IP 주소는 네트워크 세상의 "전화번호"와 같다
	ㄴ> 인터넷망에 연결된 수십억 대의 컴퓨터 중에서 내가 데이터를 보내고자 하는 정확한 
	      목적지 컴퓨터를 찾기 위해 필요하다. IP 주소가 없다면 컴퓨터들은 서로 데이터를
	      어디로 보내야 할지 알 수 없다.

3. 127.0.0.1은 무엇인가?
	ㄴ> 이 주소는 전 세계 모든 컴퓨터가 약속한 '루프백 주소'로 쉽게 말해 '자기 자신'을 
		가리키는 특수 주소이다.
		즉, 랜선을 뽑아도, 인터넷이 안되는 환경이도 이 주소는 무조건 작동한다.
		외부 네트워크로 나가지 않고, 내 컴퓨터 안에서 돌기 때문에 '루프백'이라 한다.

4. localhost는 왜 존재하는가?
	ㄴ> localhost는 숫자로된 Ip주소를 사람이 기억하기 쉽게 문자로 바꾼 '닉네임'이다.
		즉, 사람의 편의성으로 인해 생겨난 것이라고 보면 된다. (테스트 용도로도)
		다만, 로컬에서만 통하는 전용 별명이다.

[ DNS 확인 ]
1. cat /etc/resolv.conf : 네임서버가 저장되어 있는 파일 
			      해당 명령어를 통해 나오는 DNS서버를 분석한다.

2. DNS란? : 인터넷 세상의 '전화번호부'역할을 한다.
		localhost와 비슷한 역할을 하지만, DNS의 경우 외부 사이트 주소를 찾을 때 찾아가는
		역할을 해준다. (localhost는 내 컴퓨터 내부 /etc/hosts 파일에만 적힌것을 찾아줌)




[ 구글로 신호를 보내보기 ]
1. ping google.com : 구글로 핑을 보내기
2. ping -c 4 google.come : 구글로 핑을 4번 보내기
3. ping 8.8.8.8 : 구글로 핑 보내기 (IP버전)




[ DNS 조회 ]
1. sudo dnf install bind-utils -y (이미 설치된 경우 설치되어있다고 뜸!)
2. nslookup : DNS 서버에 직접 질문을 던져 특정 도메인의 IP 주소를 찾아내거나,
		 반대로 IP 주소로 도메인 이름을 조회하는 네트워크 관리 명령어이다.

3. nslookup google.com : 구글의 IP주소를 해당 명령어로 확인할 수 있다. 
				 주로 사용자와 근접한 ip가 자주 사용하는 여러개의 ip중 대표적인 주소를
				 보여준다. 

4. nslookup 8.8.8.8 : dns.google.com 이라고 주소의 DNS를 알려준다.

* 이때 구글이나 대형 웹사이트들의 경우 여러개의 IP주소를 볼 수 있을 텐데 이 이유는
  너무 많은 사람들이 몰리면 서버가 마비되버려 이용할 수 없기 때문에, 가상의 여러 ip주소들을 
  만들어서 1번 ip가 다 찬경우 자연스럽게 다음 접속자는 다음 ip로 접속되도록 한다.
 



[ 웹 요청 확인 ]
1. sudo dnf install curl -y
2. curl : 터미널 창에서 URL 주소를 이용해 서버와 데이터를 주고받을 때 사용하는 아주 강력한
	   명령줄(CLI) 데이터 전송 도구이다.
	   해당 명령어는 서버가 보내준 텍스트 원본 데이터(HTML, JSON)를 터미널에 그대로 출력한다

3. 주로 언제 사용하는가?
	ㄴ> 브라우저를 켜지 않고도 터미널 안에서 웹사이트가 잘 뜨는지 확인할때 사용한다
	ㄴ> 웹 서버가 정상이라면 해당 사이트의 HTML 소스 코드가 출력된다.
	ㄴ> 다만, 구글의 경우 현재 상태와 간단한 HTML만 표시된다.

4. curl -l : 웹사이트의 본문(HTML)은 빼고, 서버 상태 코드가 담긴 헤더 정보만 확인할때 사용한다
	     서버 디버깅 시 가장 많이 사용한다.

5. curl -L : 특정 웹사이트가 다른 주소로 자동 이동되도록 설정되어 있을 때, 그 이동 경로를 
		끝까지 쫓아가서 최종 결과물을 가져오라고 명령하는 옵션이다.

6. curl google.com : 현재 구글이 정상 작동하고 있는지 확인한다.

7. curl localhost : 현재 컴퓨터(본인)에 아파치 웹 서버가 켜져 있는지 확인한다.




[ 방화벽 확인 ]
1. sudo firewall-cmd --list-all : 현재 시스템에 적용된 방화벽의 모든 허용/차단 규칙을 보여달라!
2. sudo firewall-cmd --permanent --add-service=http : http, 즉 80번 포트의 접근을 허용한다.
								     영구적으로 (permanent) 허용한다!
								     이는 껐다 켜도 평생 유지함.

3. sudo firewall-cmd --reload : 방금 바뀐 새 규칙을 즉시 적용하라 (즉, 새로고침 및 적용)
					단, 위의 add의 경우 permanent를 적용하지 않을시 원래 값으로
					(default)값으로 돌아가게 됨. 
					permanent 선언을 하였다면, 무사히 적용됨.


* 왜 reload를 하나요? (재부팅하지 않고?)
	ㄴ> 컴퓨터를 껐다 켜도 새로 읽기 때문에 적용은 되지만, 이 순간에 방화벽 서비스를 아예 
	      껐다 킨다면 그 순간 기존에 연결되어 있던 모든 사용자의 접속도 끊겨버리기 때문에
	      reload 방식을 사용하여 사용자의 접속을 유지하면서 규칙만 업데이트를 해준다.

4. sudo firewall-cmd --add-port=80/tcp : 사실상 service=http와 같은 방식이지만, service로 추가하면
						    내부적으로 80번 포트의 접근을 허용 (보이지는 않음)한다
						    따라서 사람들은 대부분 service=http의 방식을 사용한다




[ 직접 HTML 생성하여 띄우기 ]
1. cd /var/www/html : 해당 디렉토리로 이동
2. pwd : 내가 어딘지 확인
3. ls -al : 파일 목록을 보여준다
---------------------------------------------------> 내가 지금 어디 폴더에 있는지 확인!
4. sudo nano index.html (또는 sudo vi index.html)
5. html 내용 입력하기
6. sudo cp index.html index_backup.html : 항상 백업 파일 남겨두기 (습관화)
7. 브라우저에 http://localhost 입력해보기
8. 내가 작성한 html 내용이 뜨는 것을 확인할 수 있음

	      기존에 /var/www/html 에는 아무 파일이 없었다. 즉, 디폴트 파일에 아무것도 없으니
	      그 대신 아무것도 없을때 나타나는 html 파일을 불러와 띄운거였다.
